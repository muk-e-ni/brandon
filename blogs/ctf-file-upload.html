This module explores the two types of reconnaissance: passive and active. It delves into some
tools that can be used to carry out passive reconnaissance like whois, nslookup and dig. It also
establishes foundtation knowledge on online tools such as DNSDumpster and Shondan.io.


Pasive Vs. Active Reconnaissance

Passive reconnaissance focuses on getting publically available info that can be useful
when exploiting a target. Active recon on the other hand, focuses on actively engaging or interacting
with the target to get relevant info.

nslookup and DIG

Nslookup stands for Name Server Lookup while dig stands for Domain Information Groper.
Nslookup can be used with three main parameters:

  • Options
  • Domain Name
  • Server
Query Type
  • Ipv4 addresses
  • AAAA: IPv6 addresses
  • CNAME: Canonical Name
  • MX: Mail servers
  • SOA: Start of Authority
  • TXT: TXT records
Eg. Nslookup –type=A tryhackme.com 1.1.1.1 OR nslookup – type=a Using dig: 
 dig @1.1.1.1 tryhackme.com MX/A/TXT
